## المبدأ الحاكم

> **"If the page looks crowded, move the next section to a new page."**

إذا كان آخر عنصر في الصفحة سيترك انطباعًا بصريًا مزدحمًا أو سيقترب من الفوتر بشكل غير مريح، فلا تُضغَط المحتوى ولا تُعدَّل الهوامش — انقل القسم كاملًا إلى الصفحة التالية.

---

## أولوية النظام

```
Human Layout  >  Visual Cleanliness  >  Consistency  >  Page Count
```

عدد الصفحات **ليس** مقياسًا للجودة. التنسيق الذي يبدو كأن إنسانًا صممه أهم من استغلال كل سطر متاح.

---

## الحل المعتمد: CSS-only chain

التطبيق التقني يعتمد على سلسلة CSS ناعمة فقط:

```css
/* العنوان لا يُترك وحيدًا */
.section-heading {
  break-after: avoid;
  page-break-after: avoid;
}

/* أول عنصرين من المحتوى يتبعان العنوان */
.section-body > *:first-child {
  break-before: avoid;
  page-break-before: avoid;
}
.section-body > *:nth-child(2) {
  break-before: avoid;
  page-break-before: avoid;
}
```

**لماذا CSS وليس Python guard؟**
- الأقسام الطويلة تحتوي `<ul>` واحدة كبيرة — أي guard بـ `break-inside: avoid` يقفل الـ`<ul>` كاملة مع العنوان → يجبر قسمًا طويلًا على الانتقال كاملًا للصفحة التالية بشكل قسري.
- CSS soft hints تسمح لـWeasyPrint بالكسر الطبيعي بعد أول العناصر وتُبقي التدفق إنسانيًا.
- بعد إعادة ترتيب الأقسام (commit `f20882d`)، لا يوجد heading بعد 216mm في أي صفحة.

## Human Layout Rule — منطق القوائم الذكي — commit `323493c`

> **"هل سيظن القارئ أن إنسانًا صمم هذه الصفحة؟"**
> إذا كانت الإجابة لا → انقل القائمة كاملة للصفحة التالية.

**مرجع التنفيذ الأول:** commit `b357a0e` (CHUNK_SIZE=3 — مرجع أرشيفي)
**مرجع التنفيذ النهائي:** commit `323493c` (تصنيف ذكي قصيرة/طويلة)

### القاعدة

كل قائمة تُصنَّف تلقائيًا حسب عدد عناصرها وحجم نصها:

| المعيار | النوع | التعامل |
|---------|-------|---------|
| ≤ 5 عناصر **أو** نص < 550 حرف | قصيرة | `short-list-guard` — كتلة صلبة، تنتقل كاملة إن لم تتسع |
| > 5 عناصر **و** نص ≥ 550 حرف | طويلة | `long-list-flow` — تدفق حر بين الصفحات |

في جميع الحالات: كل `li` وحدة صلبة لا تنكسر بين صفحتين.

### التطبيق التقني: `guard_list_starts()` في `convert.py`

```python
SHORT_ITEMS = 5     # قائمة بـ≤5 عناصر → قصيرة
SHORT_CHARS = 550   # قائمة بنص <550 حرف → قصيرة
```

- القائمة القصيرة → `<div class="short-list-guard">` يحتوي الـUL كاملًا.
- القائمة الطويلة → `<div class="long-list-flow">` بدون تقييد على الـUL.
- لا تطبق على الجداول (`table-page-group`).

### CSS

```css
/* القائمة القصيرة — كتلة صلبة كاملة */
.short-list-guard {
  break-inside: avoid-page;
  page-break-inside: avoid;
  display: block;
}

/* القائمة الطويلة — تدفق حر */
.long-list-flow {
  display: block;
}

/* كل li لا ينكسر في جميع الحالات */
.section-body li {
  break-inside: avoid-page;
  page-break-inside: avoid;
}
```

## متى يُعاد النظر في الـguard؟

**شرط واحد فقط:** إذا ظهر heading فعليًا بعد **250mm من أعلى الصفحة** في أي بطاقة مستقبلية.

عندها فقط:
1. أضف القسم الإشكالي يدويًا إلى `CLEAN_PAGE_SECTIONS` في `convert.py`.
2. إذا تكرر في أكثر من قسم → ادرس guard محدودًا للأقسام القصيرة جدًا (chars < 50 بلا قوائم).

**لا تُعد guard بشكل عام** — الـCSS-only chain كافٍ لكل الأقسام الطويلة.

---

## قاعدة حسابية عند وضع Section جديد

1. **احسب المساحة المتبقية** في الصفحة قبل إضافة أي قسم.
2. **المساحة المثالية لبداية قسم** = ارتفاع العنوان + أول عنصرين.
3. إذا كانت المساحة المتبقية < المساحة المثالية → **انقل القسم كاملًا للصفحة التالية**.
4. في حالات الشك → **اختر دائمًا الصفحة الأنظف**.

```
✗ ضغط الأسطر (line-height)

✗ تقليل الهوامش (margin-top / margin-bottom)

✗ محاولة ملء كل سنتيمتر من الصفحة

```

## التطبيق التقني في `convert.py`

| القاعدة | CSS المقابل |
|---------|------------|
| العنوان يبقى مع محتواه | `.section-heading { break-after: avoid }` |
| أول عنصرين يتبعان العنوان | `.section-body > *:first-child/:nth-child(2) { break-before: avoid }` |
| العنوان الفرعي h3 يبقى مع محتواه | `.section-body h3 { break-after: avoid }` |
| الفقرات لا تنكسر | `.section-body p { break-inside: avoid-page }` |
| عناصر القائمة لا تنكسر | `.section-body li { break-inside: avoid-page }` |
| الجدول الكبير + عنوانه = صفحة مستقلة | `.section.table-page-group { break-before: page; break-inside: avoid }` |
| حماية orphans | `orphans: 3; widows: 3` على `.section` و `.section-body` |
| قسم يحتاج صفحة نظيفة — استثناء يدوي | `CLEAN_PAGE_SECTIONS` في Python + `.section.clean-page-required { break-before: page }` |

## الفراغات البيضاء مقبولة

الفراغ الأبيض فوق الفوتر — حتى لو كان 15-20mm — **مقبول ومقصود** طالما أن القسم التالي سيبدأ بشكل نظيف في الصفحة التالية.

هذه الفراغات **ليست خللًا** — هي نتيجة حتمية لتوزيع محتوى ذي أحجام ثابتة على صفحات مع الحفاظ على النظافة البصرية.
