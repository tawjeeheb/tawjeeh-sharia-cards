# قفل دائم – عنصر الدورات الداعمة (Coursera Link Lock)

**تاريخ الاعتماد:** 2026-06-09
**نوع القفل:** إلزامي نهائي — يسري على جميع ملفات Tawjeeh HUB

---

## المشكلة

أي رابط Coursera يؤدي إلى:
- `ERR_CONNECTION_RESET`
- صفحة خطأ
- Redirect غير صحيح
- صفحة تسجيل مؤقتة
- رابط بحث
- رابط Session منتهي

يُعتبر رابطًا فاشلًا **مرفوضًا**.

---

## القواعد الإلزامية

### 1. يُمنع استخدام

- روابط البحث
- روابط session
- روابط enroll المؤقتة
- روابط promo
- روابط track
- روابط login
- روابط sessionId
- روابط query string الطويلة

### 2. الرابط المقبول فقط

```
https://www.coursera.org/learn/slug-name
https://www.coursera.org/professional-certificates/slug-name
https://www.coursera.org/specializations/slug-name
```

**بدون أي معاملات إضافية (`?` أو `&`).**

---

## التحقق الإجباري قبل الاعتماد

قبل اعتماد أي دورة، افتح الرابط النهائي.

إذا لم يفتح مباشرة على صفحة الدورة الرسمية وتعرض:

```
اسم الدورة + اسم الجهة المقدمة + زر التسجيل
```

→ يُرفض الرابط فورًا ويُبحث عن الرابط الرسمي الصحيح.

---

## ممنوع نهائيًا

```
✗ نسخ الرابط من نتائج البحث
✗ نسخ الرابط من Session شخصية
✗ نسخ الرابط من صفحة تسجيل الدخول
✗ نسخ الرابط بعد الضغط على Enroll
✗ نسخ رابط يحوي أي query parameters
```

---

## آلية الاختبار الإجبارية

كل دورة يجب أن تجتاز **جميع** الشروط التالية:

| الشرط | المطلوب |
|-------|---------|
| HTTP Status | 200 |
| نوع الصفحة | صفحة دورة رسمية |
| الـSlug | ثابت، لا يتغير |
| نافذة خاصة | يفتح في Incognito |
| بدون تسجيل دخول | يفتح دون حساب |

إذا فشل أي شرط → **تُستبدل الدورة بدورة أخرى**.

---

## حكم المخالفة

> أي رابط Coursera لا يجتاز هذه الشروط يُعتبر **خطأً إنتاجيًا جسيمًا** ويُمنع استخدامه في جميع ملفات Tawjeeh HUB مستقبلًا.
>
> لا يُسمح بإخراج أي دورة قبل اجتياز هذا الاختبار.
