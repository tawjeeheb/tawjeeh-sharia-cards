# SOURCE_EVIDENCE_LOCK

**الإصدار:** 1.0  
**تاريخ الاعتماد:** 2026-06-09  
**الحالة:** مُفعَّل — لا يُعدَّل إلا بموافقة صريحة من المستخدم

---

## القاعدة الذهبية

> **أي معلومة مهنية بلا دليل = لا تُذكر.**

---

## المعلومات التي تحتاج دليلًا

كل معلومة من القائمة التالية يجب أن يكون لها دليل قابل للتتبع قبل إدراجها في أي بطاقة:

| المعلومة | مثال على الدليل المطلوب |
|---------|------------------------|
| وجود المهنة | كود SSC في دليل التصنيف السعودي الموحد للمهن |
| التصنيف الوطني | صفحة أو PDF رسمي من هيئة تنمية الموارد البشرية |
| الجهات التي تمارس المهنة | إعلان وظيفي أو موقع الجهة الرسمي |
| شروط المؤهل | نظام أو لائحة أو دليل قبول رسمي |
| الشروط النظامية | مرسوم ملكي أو قرار وزاري أو نظام منشور رسميًا |
| الراتب إذا ذُكر برقم | تقرير رسمي أو إعلان وظيفي موثق أو تقرير سوق العمل |
| برامج التأهيل | صفحة البرنامج الرسمية على موقع المؤسسة |
| الشهادات المهنية | صفحة الشهادة الرسمية على موقع الجهة المانحة |
| الدورات | صفحة الدورة المباشرة على منصة التدريب |
| جهات التوظيف | إعلان وظيفي سابق أو موقع الجهة الرسمي |
| طرق التقديم | منصة التقديم الرسمية أو صفحة التوظيف |
| المتطلبات المهنية | وثيقة رسمية من الجهة المنظِّمة للمهنة |
| أي ادعاء عن السوق السعودي | مصدر سعودي موثوق — حكومي أو أكاديمي أو إعلامي رسمي |

---

## المصادر المقبولة

| الأولوية | نوع المصدر | مثال |
|----------|-----------|------|
| 1 | موقع جهة حكومية أو شبه حكومية | وزارة العدل، هيئة المحامين السعوديين، النيابة العامة |
| 2 | نظام أو لائحة أو دليل رسمي | نظام القضاء م/78، نظام المحاماة م/38، دليل SSC |
| 3 | صفحة برنامج رسمية | صفحة الدبلوم على موقع معهد الإدارة العامة مباشرة |
| 4 | صفحة شهادة رسمية | صفحة CAMS على موقع ACAMS مباشرة |
| 5 | إعلان وظيفي موثق | إعلان على جدارة أو الموقع الرسمي للجهة |
| 6 | تقرير رسمي أو PDF رسمي | تقرير سوق العمل من هيئة تنمية الموارد البشرية |
| 7 | منصة تدريب رسمية | إثرائي، Coursera، رواق، edX |
| 8 | خبر موثوق | مصدر إعلامي معترف به عند غياب مصدر مباشر |

---

## المصادر غير الكافية وحدها

تُرفض المعلومة إذا كان مصدرها الوحيد أحد ما يلي:

- منشور عام بلا جهة محددة أو تاريخ
- صفحة رئيسية عامة لموقع دون رابط مباشر للمعلومة
- رابط بحث (نتائج Google أو Bing)
- كلام تسويقي بلا تفاصيل قابلة للتحقق
- مصدر غير سعودي لإثبات ممارسة تخص السوق السعودي
- رابط لا يفتح أو يُعيد توجيهًا لصفحة خطأ
- رابط يشترط تسجيل دخول للوصول إلى المعلومة

## اختبار القبول

قبل اعتماد أي معلومة، يجيب النظام على الأسئلة الأربعة:

| السؤال | إذا لم توجد إجابة |
|--------|------------------|
| ما الدليل؟ | تُحذف المعلومة |
| أين الرابط أو المرجع؟ | تُحذف المعلومة |
| هل يُثبت الدليل نفس المعلومة تحديدًا؟ | تُحذف المعلومة |
| هل المصدر مناسب للسياق السعودي؟ | تُحذف المعلومة |

---

## تطبيق خاص — العناصر التي تتطلب دليلًا إلزاميًا

لا يُسمح بإدراج أي مما يلي إلا مع رابط أو دليل واضح في ملف التدقيق أو داخل البطاقة:

| العنصر | موضع توثيق الدليل |
|--------|------------------|
| برنامج تأهيل | رابط مباشر لصفحة البرنامج داخل البطاقة |
| شهادة مهنية | رابط مباشر لصفحة الشهادة داخل البطاقة |
| دورة داعمة | رابط مباشر لصفحة الدورة داخل البطاقة |
| جهة توظيف | اسم الجهة + إشارة لإعلان سابق أو موقع رسمي |
| شرط مهني نظامي | رقم النظام أو المرسوم في ملف التدقيق |

## العلاقة مع القفول الأخرى

| القفل | نقطة التقاطع |
|-------|-------------|
| LINK_VALIDATION_LOCK | يتحقق من صحة الروابط التقنية — SOURCE_EVIDENCE_LOCK يتحقق من ملاءمة المصدر للمعلومة |
| QUALITY_GATE | المرحلة الثانية والثالثة والرابعة تفترض أن كل عنصر مدعوم بدليل وفق هذا القفل |
| CONTENT_REGISTRY | كل عنصر مسجل يجب أن يكون له دليل اصلي موثق في وقت تسجيله |
| ROLE_SPECIFIC_CONTENT_LOCK | التخصص المهني وحده لا يكفي — يجب أن يكون التخصص مدعومًا بأدلة لا بافتراضات |
