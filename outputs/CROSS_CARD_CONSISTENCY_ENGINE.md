## 1. ترتيب العناصر

**المعيار:** 18 قسمًا بالترتيب المحدد في FINAL_CARD_APPROVAL_CHECKLIST.

**الفحص:** هل تظهر الأقسام الـ 18 بنفس الترتيب في البطاقة المفحوصة؟

| الحالة | الحكم |
|--------|-------|
| الترتيب مطابق | مقبول |
| قسم مقدَّم أو مؤخَّر | مرفوض — يعود للتصحيح |
| قسم مفقود | مرفوض — يعود للإكمال |

## 2. أسلوب كتابة الفقرات

**المعيار (من بطاقة قاضي):**
- **طبيعة العمل:** جملة وصفية واحدة متكاملة تعرّف الدور وتذكر بيئته والمنظومة التي ينتمي إليها
- **الخبرات المطلوبة:** فقرة نثرية واحدة مدمجة لا قائمة نقطية
- **المسار الوظيفي:** ثلاثة محاور مسماة (مسار الترقي / مسار الانتقال / مسار التخصص)

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| طبيعة العمل جملة واحدة متكاملة | مقبول |
| طبيعة العمل نقاط أو فقرات متعددة | مرفوض |
| الخبرات فقرة نثرية | مقبول |
| الخبرات قائمة نقطية | مرفوض |
| المسار يحتوي المحاور الثلاثة | مقبول |
| محور مفقود أو مضاف | مرفوض |

---

## 3. عدد النقاط في كل عنصر

**المعيار (من بطاقة قاضي):**

| العنصر | العدد المعتمد |
|--------|--------------|
| المهام الرئيسية | 5 نقاط |
| المزايا الوظيفية | 4–6 نقاط لكل قطاع |
| المهارات المطلوبة | 5 مهارات |
| الدورات الداعمة | 5 دورات |
| الشهادات المهنية | 4 شهادات |
| برامج التأهيل | 3 برامج |
| الملاحظات المهنية | 3–5 ملاحظات |
| النصائح العملية | 4–6 نصائح |

**الفحص:** انحراف بنقطة واحدة مقبول. انحراف بنقطتين أو أكثر يستوجب المراجعة.

| الحالة | الحكم |
|--------|-------|
| العدد مطابق أو بفارق ±1 | مقبول |
| الفارق 2 أو أكثر | مرفوض — يراجع المحتوى |

---

## 4. أسلوب كتابة الرواتب

**المعيار (من بطاقة قاضي):**
```
- [المسمى الوظيفي]: نحو [X,000] – [Y,000] ريال تقريبًا.
```
- الوحدة دائمًا «ريال»
- كلمة «نحو» في البداية وكلمة «تقريبًا» في النهاية
- النطاق بصيغة X,000 – Y,000
- آخر درجة تُكتب بصيغة «من X,000 ريال فأعلى تقريبًا»

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| الصيغة مطابقة | مقبول |
| غياب «نحو» أو «تقريبًا» | مرفوض |
| استخدام «SAR» أو «$» أو «ألف» | مرفوض |
| رقم بدون نطاق (رقم واحد فقط) | مرفوض إلا للدرجة الأعلى |

---

## 5. أسلوب كتابة القطاعات

**المعيار (من بطاقة قاضي):**
- كل قطاع يُفتح بـ `<p class="sector-label">اسم القطاع</p>`
- الشروط تُكتب بصيغة `<span class="condition-label">العنوان:</span> النص`
- القطاعات لا تُدمج — كل قطاع كتلة مستقلة

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| sector-label موجود لكل قطاع | مقبول |
| قطاعان في كتلة واحدة بدون sector-label | مرفوض |
| شروط بدون condition-label | مرفوض |

---

## 6. أسلوب كتابة الشهادات

**المعيار (من بطاقة قاضي):**
```
* [اسم الشهادة الكامل (الاختصار)](رابط): وصف وظيفي يربطها بالدور (غير مدعومة)
```
- الاسم الكامل أولًا ثم الاختصار بين قوسين
- الرابط مباشر لصفحة الشهادة
- الوصف يذكر مهارة أو نشاطًا يمارسه الدور
- ينتهي الوصف بـ `(غير مدعومة)` بين قوسين

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| الصيغة مطابقة | مقبول |
| غياب «(غير مدعومة)» | مرفوض |
| الاختصار بدون الاسم الكامل | مرفوض |
| وصف عام لا يذكر ما يفعله هذا الدور | مرفوض |

---

## 7. أسلوب كتابة الدورات

**المعيار (من بطاقة قاضي):**
```
* دورة [اسم الدورة العربي القارئي](رابط): تدعم مهارة [اسم المهارة من قسم المهارات]
```
- كلمة «دورة» في البداية
- الاسم العربي القارئي (وفق READER_FIRST_COURSE_TITLE_LOCK)
- الرابط مباشر
- الربط يذكر المهارة بنفس صياغتها في قسم «المهارات المطلوبة»

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| الصيغة مطابقة | مقبول |
| غياب كلمة «دورة» | مرفوض |
| الربط بمهارة غير مُدرجة في قسم المهارات | مرفوض |
| اسم الدورة إنجليزي أو ترجمة حرفية جامدة | مرفوض |

---

## 8. أسلوب كتابة البرامج

**المعيار (من بطاقة قاضي):**
```
* [اسم البرنامج](رابط) - اسم المؤسسة: وصف يشرح دوره في تأهيل هذه المهنة تحديدًا
```
- الاسم أولًا مع الرابط، ثم شرطة، ثم اسم المؤسسة، ثم نقطتان، ثم الوصف
- الوصف يذكر الفجوة التي يسدها لهذه المهنة
- لا يكتفي بوصف البرنامج العام

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| الصيغة مطابقة | مقبول |
| وصف عام لا يذكر فجوة المهنة | مرفوض |
| غياب اسم المؤسسة | مرفوض |

---

## 9. أسلوب كتابة الملاحظات المهنية

**المعيار (من بطاقة قاضي):**
- كل ملاحظة فقرة نثرية كاملة (لا جملة قصيرة)
- تنتهي بمرجع صريح: «ويستند ذلك إلى [المصدر]»
- الملاحظة تكشف معلومة غير واضحة في الوصف العام للمهنة

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| ملاحظة بمرجع صريح | مقبول |
| ملاحظة بلا مرجع | مرفوض |
| ملاحظة عامة تصلح لأي مهنة | مرفوض |
| ملاحظة أقصر من جملتين | مرفوض |

---

## 10. أسلوب كتابة النصائح العملية

**المعيار (من بطاقة قاضي):**
- كل نصيحة فقرة نثرية متكاملة تبدأ بفعل أمر مباشر
- تُفسّر «لماذا» بجملة تبدأ بـ «لأن» أو «إذ»
- النصيحة تعكس تحديًا حقيقيًا لهذه المهنة تحديدًا
- للتخصصات الدقيقة: تُضاف جملة تطبيقية إضافية تبدأ بـ «للتخصصات الدقيقة:» إن وجد تمييز مهم

**الفحص:**

| الحالة | الحكم |
|--------|-------|
| النصيحة تبدأ بفعل أمر وتشرح السبب | مقبول |
| نصيحة بدون «لأن» أو «إذ» | مرفوض |
| نصيحة عامة تصلح لأي خريج | مرفوض |

## آلية الفحص

**عند مراجعة بطاقة جديدة قبل الاعتماد:**

1. ضع البطاقة المفحوصة جنبًا إلى جنب مع بطاقة قاضي
2. مرّ على البنود العشرة بالترتيب
3. سجّل كل بند بـ ✓ أو ✗
4. أي ✗ واحد = البطاقة مرفوضة حتى تُصحَّح

**موضع هذا المحرك في المنظومة:**  
يُطبَّق بعد QUALITY_GATE وقبل FINAL_CARD_APPROVAL_CHECKLIST.
